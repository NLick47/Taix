# v1.2.7 更新日志

## 主要变化

- 客户端开放休眠判定相关设置
- 多轨道图优化，色块加入悬浮提示并且可以和时间轴高亮联动
- 安装器利用webview实现gui，和一些方面的修复及优化
- windows端统计记时采用轮询方案进行重构，避免特殊情况造成焦点事件丢失
- 调色板采用自绘方案实现会更加贴合ui，不再进行弹窗
- 通过对平台统计端调整，和服务端合并算法优化增加时间轴可读性减少碎片化

## 优化与修复

- 增加2种窗口渐变方案，修复渐变配置无法持久化
- 修复保存窗口大小开关显示问题
- 服务端修复删除网站分类后未清除规则缓存
- 分类图标选择不再占用图片缓存容量
- 修复计划任务启动时托盘图标有概率消失
- 体积优化 - 客户端采用比以往版本更加激进的裁剪配置、后台服务各组件也进行了精简
- 时间轴色块悬浮优化
- 修复迁移时分类id为0的孤立数据 
- 修复柱状图分类徽章显示策略不统一
- 修复客户端未正确区分"精确匹配忽略"和"通配符模式匹配忽略"，无实际功能影响应用详情页会多出一条提示
- 优化windows平台图标提取，失败时尝试更多提取路径
- 修复从详情页返回后框选时间轴数据丢失的问题
- 优化右键点击忽略应用/站点后能够立即刷新当前页面
- 优化详情页修改数据后返回能够自动刷新
- 修复部分切换Tab后进入详情页再返回出现右键Context附加混乱（未完全修复）
- 修复保存窗口大小无法恢复至最大化状态
- macos 实现monitor单例锁，windows monitor 修复单例锁无效
- 修复应用详情页右键菜单文字错误：启动应用误显示为"在浏览器打开网站"
- 优化Popup能根据窗口焦点消失
- 日期选择加入圆点用于区分选中日期

## 下载与安装

本次发布提供以下产物，按需选择

| 产物 | 文件名 | 适用场景 |
| --- | --- | --- |
| Windows 安装器 | `Taix-1.2.7-Setup.exe` | 推荐普通用户使用 |
| Windows 绿色版 | `Taix-1.2.7-win-x64.zip` | 便携使用，需手动执行脚本注册开机自启（任务计划）服务 |
| macOS | `Taix-1.2.7-macos-arm64.dmg` | Apple Silicon 设备，macOS 12.0+ |

### Windows 安装器（推荐）

1. 下载 `Taix-1.2.7-Setup.exe`
2. 双击运行，按引导完成安装
3. 安装器会自动注册服务并启动，无需手动操作

> 💡 **从绿色版切换到安装器**：若想从绿色版切换到安装器，只需指定安装器得安装目录为绿色版exe文件所在得文件夹，安装器会自动完成服务停止和文件替换

### Windows 绿色版

1. 下载并解压 `Taix-1.2.7-win-x64.zip` 到任意目录（建议非系统盘）
2. 以**管理员身份**打开 PowerShell，进入解压目录
3. 运行 `install.ps1` 完成服务注册（脚本成功后会自动启动相关服务）
4. 卸载时同样以管理员身份运行 `uninstall.ps1`

> ⚠️ **重要**：`install.ps1` 会创建任务计划并指向当前解压目录，因此绿色版**安装后请勿移动或重命名该目录**，否则任务计划指向的路径失效将导致开机自启失败。若必须迁移位置，将整个目录移动到新位置后，直接在新目录重新运行 `install.ps1` 即可（脚本会强制覆盖原任务计划并刷新指向路径）

> 绿色版升级：若从 1.2.0 及之后版本升级，可解压后直接替换 exe；若更换了安装目录，在新目录运行 `install.ps1` 即可覆盖注册，请保留原图标目录与数据目录

### macOS

DMG 内含 `Taix.app`（客户端）和 `TaixTools/`（后台服务 + 安装脚本），两者都需要部署

1. 将 `Taix.app` 和 `TaixTools` 一起拖入 `Applications`
2. 终端运行 `/Applications/TaixTools/install-launchagent.sh` 注册后台服务
3. 启动 `Taix.app`

卸载运行 `/Applications/TaixTools/uninstall.sh`，Gatekeeper 拦截可在「系统设置 → 隐私与安全性」放行


## 注意事项

**1. 绿色版笔记本用户升级**

> ⚠️ **重要**：本次版本对计划任务做了调整以提升笔记本上的稳定性。若你使用的是**绿色版**且当前版本在 1.2.0 - 1.2.5 之间，升级到本版本后请重新执行一次 `install.ps1` 刷新任务计划，避免未能得到及时得问题修复

**2. 时间格式变更说明**

> ⚠️ **重要**：自 v1.2.0 版本起，SQLite 本地存储已统一改为 UTC 时间，迁移后的数据将不再兼容旧版客户端，会存在时间偏移

**3. 从 1.2.0 之前的版本升级**

或其它迁移中遇到的问题，暂时移步到 #12

**4. 启动时间说明**

> ⚠️ **重要**：1.2.6版本已针对冷启动优化，自测冷启动在 0.5 秒左右。若启动时间明显偏长，可能是腾讯电脑管家等安全类软件扫描拦截所致，可将其加入对应软件的信任白名单
